# Test Plan: AwesomeQA Registration & Cart
**Author:** [Basel Badr]
**Status:** Completed

### 1. Introduction
This plan covers the functional and security testing of the user registration and shopping cart modules on the AwesomeQA demo site.

### 2. Features to be Tested
* User Registration (Positive/Negative/Boundary)
* Shopping Cart Logic (Inventory sync and quantity limits)
* Basic Security (Password strength and masking)

### 3. Pass/Fail Criteria
* **Pass:** All high-priority bugs are resolved and core functionality works as expected.
* **Fail:** Any security vulnerability or data integrity issue (e.g., negative quantities) remains open.
